このページでは、Model Context Protocol (MCP)の機能と汎用性を示す様々なMCPサーバーを紹介します。これらのサーバーにより、大規模言語モデル(LLM)はツールやデータソースに安全にアクセスできます。

## リファレンス実装

これらの公式リファレンスサーバーは、MCPのコア機能とSDKの使用方法を示しています:

### データとファイルシステム
- **[Filesystem](https://github.com/modelcontextprotocol/servers/tree/main/src/filesystem)** - 設定可能なアクセス制御を備えた安全なファイル操作
- **[PostgreSQL](https://github.com/modelcontextprotocol/servers/tree/main/src/postgres)** - スキーマ検査機能を備えた読み取り専用データベースアクセス
- **[SQLite](https://github.com/modelcontextprotocol/servers/tree/main/src/sqlite)** - データベース操作とビジネスインテリジェンス機能
- **[Google Drive](https://github.com/modelcontextprotocol/servers/tree/main/src/gdrive)** - Google Driveのファイルアクセスと検索機能

### 開発ツール
- **[Git](https://github.com/modelcontextprotocol/servers/tree/main/src/git)** - Gitリポジトリの読み取り、検索、操作ツール
- **[GitHub](https://github.com/modelcontextprotocol/servers/tree/main/src/github)** - リポジトリ管理、ファイル操作、GitHub API統合
- **[GitLab](https://github.com/modelcontextprotocol/servers/tree/main/src/gitlab)** - プロジェクト管理を可能にするGitLab API統合
- **[Sentry](https://github.com/modelcontextprotocol/servers/tree/main/src/sentry)** - Sentry.ioからの問題取得と分析

### Webとブラウザ自動化
- **[Brave Search](https://github.com/modelcontextprotocol/servers/tree/main/src/brave-search)** - BraveのSearch APIを使用したWebおよびローカル検索
- **[Fetch](https://github.com/modelcontextprotocol/servers/tree/main/src/fetch)** - LLM使用に最適化されたWebコンテンツ取得と変換
- **[Puppeteer](https://github.com/modelcontextprotocol/servers/tree/main/src/puppeteer)** - ブラウザ自動化とWebスクレイピング機能

### 生産性とコミュニケーション
- **[Slack](https://github.com/modelcontextprotocol/servers/tree/main/src/slack)** - チャンネル管理とメッセージング機能

- **[Memory](https://github.com/modelcontextprotocol/servers/tree/main/src/memory)** - 知識グラフベースの永続メモリシステム

### AIと専門ツール
- **[EverArt](https://github.com/modelcontextprotocol/servers/tree/main/src/everart)** - 様々なモデルを使用したAI画像生成
- **[Sequential Thinking](https://github.com/modelcontextprotocol/servers/tree/main/src/sequentialthinking)** - 思考シーケンスによる動的問題解決
- **[AWS KB Retrieval](https://github.com/modelcontextprotocol/servers/tree/main/src/aws-kb-retrieval-server)** - Bedrock Agent Runtimeを使用したAWSナレッジベースからの検索

## はじめに

### リファレンスサーバーの使用

TypeScriptベースのサーバーは`npx`で直接使用できます:

```bash
npx -y @modelcontextprotocol/server-memory
```

Pythonベースのサーバーは`uvx`（推奨）または`pip`で使用できます:

```bash
# uvxを使用
uvx mcp-server-git

# pipを使用
pip install mcp-server-git
python -m mcp_server_git
```

### Claudeでの設定

ClaudeでMCPサーバーを使用するには、設定に追加します:

```json
{
  "mcpServers": {
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/files"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "<YOUR_TOKEN>"
      }
    }
  }
}
```
