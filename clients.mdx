## クライアント詳細

### Claudeデスクトップアプリ
ClaudeデスクトップアプリケーションはMCPを包括的にサポートし、ローカルツールやデータソースとの深い統合を可能にします。

**主な機能:**
- リソースの完全サポートにより、ローカルファイルやデータの添付が可能
- プロンプトテンプレートのサポート
- コマンドやスクリプト実行のためのツール統合
- プライバシーとセキュリティを強化するローカルサーバー接続

> ⓘ 注: Claude.aiウェブアプリケーションは現在MCPをサポートしていません。MCP機能はデスクトップアプリケーションでのみ利用可能です。

### Claude Code
Claude CodeはAnthropicのインタラクティブなエージェント型コーディングツールで、自然言語コマンドを通じてより速くコーディングするのを支援します。プロンプトとツールのMCP統合をサポートし、他のクライアントと統合するためのMCPサーバーとしても機能します。

**主な機能:**
- MCPサーバー用のツールとプロンプトサポート
- 他のMCPクライアントと統合するための独自ツールをMCPサーバーとして提供

### 5ire
[5ire](https://github.com/nanbingxyz/5ire)はMCPサーバーを通じてツールをサポートするオープンソースのクロスプラットフォームデスクトップAIアシスタントです。

**主な機能:**
- 組み込みのMCPサーバーを迅速に有効化/無効化可能
- 設定ファイルを変更することで追加サーバーを追加可能
- オープンソースでユーザーフレンドリー、初心者に適している
- MCPサポートは今後継続的に改善予定

### BeeAI Framework
[BeeAI Framework](https://i-am-bee.github.io/beeai-framework)は、強力なエージェントワークフローを大規模に構築、展開、提供するためのオープンソースフレームワークです。このフレームワークには、MCPサーバーをエージェントワークフローに簡単に統合するためのネイティブ機能である**MCP Tool**が含まれています。

**主な機能:**
- エージェントワークフローにMCPツールをシームレスに組み込み
- 接続されたMCPクライアントからフレームワークネイティブツールを迅速にインスタンス化
- エージェントMCP機能の将来サポートを計画中

**詳細情報:**
- [エージェントワークフローでMCPツールを使用する例](https://i-am-bee.github.io/beeai-framework/#/typescript/tools?id=using-the-mcptool-class)

### Cline
[Cline](https://github.com/cline/cline)はVS Code内で動作する自律型コーディングエージェントで、ファイル編集、コマンド実行、ブラウザ使用などを各ステップでユーザーの許可を得ながら行います。

**主な機能:**
- 自然言語でツールを作成・追加可能（例：「ウェブ検索するツールを追加」）
- Clineが作成するカスタムMCPサーバーを`~/Documents/Cline/MCP`ディレクトリ経由で他者と共有可能
- 設定済みMCPサーバーとそのツール、リソース、エラーログを表示

### Continue
[Continue](https://github.com/continuedev/continue)はオープンソースのAIコードアシスタントで、すべてのMCP機能を組み込みでサポートしています。

**主な機能**
- "@"でMCPリソースに言及可能
- プロンプトテンプレートがスラッシュコマンドとして表示
- 組み込みツールとMCPツールの両方をチャットで直接使用可能
- VS CodeとJetBrains IDEをサポート、あらゆるLLMと連携

### Cursor
[Cursor](https://docs.cursor.com/advanced/model-context-protocol)はAIコードエディターです。

**主な機能**:
- Cursor ComposerでMCPツールをサポート
- STDIOとSSEの両方をサポート

### Emacs Mcp
[Emacs Mcp](https://github.com/lizqwerscott/mcp.el)はMCPサーバーとインターフェースするように設計されたEmacsクライアントで、シームレスな接続と相互作用を可能にします。[gptel](https://github.com/karthink/gptel)や[llm](https://github.com/ahyatt/llm)などのAIプラグイン向けにMCPツール呼び出しをサポートし、Emacsの標準ツール呼び出し形式に準拠しています。この統合により、Emacsエコシステム内のAIツールの機能が強化されます。

**主な機能:**
- Emacs向けMCPツールサポートを提供

### fast-agent
[fast-agent](https://github.com/evalstate/fast-agent)はPythonエージェントフレームワークで、AnthropicおよびOpenAIモデルの完全なマルチモーダルサポートを備えた、エージェントとワークフローを作成するためのシンプルな宣言的サポートを提供します。

**主な機能:**
- MCPネイティブタイプに基づくPDFおよび画像サポート
- パススルーおよび再生シミュレーターを含む、エージェントアプリケーションを開発・診断するためのインタラクティブフロントエンド
- 「効果的なエージェントの構築」ワークフローの組み込みサポート
- MCPサーバーとしてエージェントを展開

### Genkit
[Genkit](https://github.com/firebase/genkit)はアプリケーションにGenAI機能を構築・統合するためのクロス言語SDKです。[genkitx-mcp](https://github.com/firebase/genkit/tree/main/js/plugins/mcp)プラグインは、クライアントとしてMCPサーバーを利用したり、GenkitツールとプロンプトからMCPサーバーを作成したりすることを可能にします。

**主な機能:**
- ツールとプロンプトのクライアントサポート（リソースは部分的にサポート）
- GenkitのDev UIプレイグラウンドでのリッチなディスカバリー
- Genkitの既存ツールとプロンプトとのシームレスな相互運用性
- 主要プロバイダーの多様なGenAIモデルに対応

### GenAIScript
[GenAIScript](https://microsoft.github.io/genaiscript/)（JavaScript）を使用してLLM向けプロンプトをプログラム的に組み立てます。JavaScriptでLLM、ツール、データをオーケストレーションします。

**主な機能:**
- プロンプトを操作するためのJavaScriptツールボックス
- 簡単で生産的な作業のための抽象化
- Visual Studio Codeとのシームレスな統合

### Goose
[Goose](https://github.com/block/goose)はオープンソースのAIエージェントで、コーディングタスクを自動化することでソフトウェア開発を強化します。

**主な機能:**
- ツールを通じてGooseにMCP機能を公開
- MCPは[拡張機能ディレクトリ](https://block.github.io/goose/v1/extensions/)、CLI、またはUIから直接インストール可能
- [独自のMCPサーバーを構築](https://block.github.io/goose/docs/tutorials/custom-extensions)することで機能を拡張可能
- 開発、ウェブスクレイピング、自動化、メモリ、JetBrainsおよびGoogleドライブとの統合用の組み込みツールを含む

### LibreChat
[LibreChat](https://github.com/danny-avila/LibreChat)はオープンソースのカスタマイズ可能なAIチャットUIで、複数のAIプロバイダーをサポートし、現在MCP統合も含んでいます。

**主な機能:**
- [コードインタープリター](https://www.librechat.ai/docs/features/code_interpreter)や画像生成ツールを含む現在のツールエコシステムをMCPサーバーを通じて拡張
- 主要プロバイダーの様々なLLMを使用したカスタマイズ可能な[エージェント](https://www.librechat.ai/docs/features/agents)にツールを追加
- オープンソースでセルフホスティング可能、安全なマルチユーザーサポート
- 将来のロードマップには拡張MCP機能サポートが含まれる

### mcp-agent
[mcp-agent]はModel Context Protocolを使用してエージェントを構築するためのシンプルで構成可能なフレームワークです。

**主な機能:**
- MCPサーバーの自動接続管理
- 複数サーバーからのツールをLLMに公開
- [効果的なエージェントの構築](https://www.anthropic.com/research/building-effective-agents)で定義されたすべてのパターンを実装
- 人間のフィードバックを待つなど、ワークフローの一時停止/再開シグナルをサポート

### Microsoft Copilot Studio
[Microsoft Copilot Studio]は、カスタムAI駆動アプリケーションとインテリジェントエージェントを構築するための堅牢なSaaSプラットフォームで、開発者が高度なAIソリューションを作成、展開、管理できるようにします。

**主な機能:**
- MCPツールのサポート
- MCPサーバーでCopilot Studioエージェントを拡張
- Microsoftの統合、管理、安全なAPI管理ソリューションを活用

### oterm
[oterm]はOllama向けのターミナルクライアントで、ユーザーがツールと接続したチャット/エージェントを作成できます。

**主な機能:**

 - ツールと接続したOllamaとの複数の完全にカスタマイズ可能なチャットセッションをサポート
 - MCPツールのサポート

### Roo Code
[Roo Code](https://roocode.com)はMCPを通じてAIコーディングアシスタンスを可能にします。

**主な機能:**
- MCPツールとリソースのサポート
- 開発ワークフローとの統合
- 拡張可能なAI機能

### Sourcegraph Cody
[Cody](https://openctx.org/docs/providers/modelcontextprotocol)はSourcegraphのAIコーディングアシスタントで、OpenCTXを通じてMCPを実装しています。

**主な機能:**
- MCPリソースのサポート
- Sourcegraphのコードインテリジェンスとの統合
- 抽象化レイヤーとしてOpenCTXを使用
- 追加MCP機能の将来サポートを計画中

### SpinAI
[SpinAI](https://spinai.dev)は、観測可能なAIエージェントを構築するためのオープンソースTypeScriptフレームワークです。このフレームワークはネイティブMCP互換性を提供し、エージェントがMCPサーバーやツールとシームレスに統合できるようにします。

**主な機能:**
- AIエージェント向け組み込みMCP互換性
- オープンソースTypeScriptフレームワーク
- 観測可能なエージェントアーキテクチャ
- MCPツール統合のネイティブサポート

### Superinterface
[Superinterface](https://superinterface.ai)はAIインフラストラクチャおよび開発者プラットフォームで、MCPサポート、インタラクティブコンポーネント、クライアントサイド関数呼び出しなどを備えたアプリ内AIアシスタントを構築します。

**主な機能:**
- Reactコンポーネントまたはスクリプトタグ経由で埋め込まれたアシスタントでMCPサーバーからのツールを使用
- SSEトランスポートサポート
- あらゆるAIプロバイダー（OpenAI、Anthropic、Ollamaなど）のあらゆるAIモデルを使用

### TheiaAI/TheiaIDE
[Theia AI](https://eclipsesource.com/blogs/2024/10/07/introducing-theia-ai/)はAI強化ツールとIDEを構築するためのフレームワークです。[AIパワードTheia IDE](https://eclipsesource.com/blogs/2024/10/08/introducting-ai-theia-ide/)はTheia AI上に構築されたオープンで柔軟な開発環境です。

**主な機能:**
- **ツール統合**: Theia AIは、Theia IDE内のAIエージェントを含め、シームレスなツール相互作用のためにMCPサーバーを利用できるようにします。
- **カスタマイズ可能なプロンプト**: Theia IDEはユーザーがプロンプトを定義・適応させ、MCPサーバーを動的に統合してワークフローを調整できます。
- **カスタムエージェント**: Theia IDEはMCP機能を活用するカスタムエージェントの作成をサポートし、ユーザーが専用ワークフローをその場で設計できます。

Theia AIとTheia IDEのMCP統合はユーザーに柔軟性を提供し、MCPを探索・適応するための強力なプラットフォームとなります。

**詳細情報:**
- [Theia IDEとTheia AI MCP発表](https://eclipsesource.com/blogs/2024/12/19/theia-ide-and-theia-ai-support-mcp/)
- [AIパワードTheia IDEをダウンロード](https://theia-ide.org/)

### Windsurf Editor
[Windsurf Editor](https://codeium.com/windsurf)は、AIアシスタンスと開発者ワークフローを組み合わせたエージェント型IDEです。革新的なAI Flowシステムを特徴とし、開発者の制御を維持しながら、協調的かつ独立したAI相互作用を可能にします。

**主な機能:**
- 人間-AIコラボレーションのための革命的なAI Flowパラダイム
- インテリジェントなコード生成と理解
- マルチモデルサポートを備えた豊富な開発ツール

### Witsy

**主な機能:**
- 複数のMCPサーバーサポート
- コマンドやスクリプト実行のためのツール統合
- プライバシーとセキュリティを強化するローカルサーバー接続
- Smithery.aiからの簡単インストール
- macOS、Windows、Linux向けオープンソース

### Zed
[Zed](https://zed.dev/docs/assistant/model-context-protocol)は高性能コードエディターで、プロンプトテンプレートとツール統合に焦点を当てた組み込みMCPサポートを備えています。

**主な機能:**
- プロンプトテンプレートがエディター内でスラッシュコマンドとして表示
- 強化されたコーディングワークフローのためのツール統合
- エディター機能とワークスペースコンテキストとの緊密な統合
- MCPリソースはサポートしていない

### OpenSumi
[OpenSumi](https://github.com/opensumi/core)はAIネイティブIDE製品を迅速に構築するのに役立つフレームワークです。

**主な機能:**
- OpenSumiでMCPツールをサポート
- 組み込みIDE MCPサーバーとカスタムMCPサーバーをサポート

### Daydreams
[Daydreams](https://github.com/daydreamsai/daydreams)はオンチェーンで何でも実行するための生成型エージェントフレームワークです

**主な機能:**
- 設定内でMCPサーバーをサポート
- MCPクライアントを公開

### Apify MCP Tester

[Apify MCP Tester](https://github.com/apify/tester-mcp-client)は、Server-Sent Events (SSE)を使用して任意のMCPサーバーに接続するオープンソースクライアントです。
これはスタンドアロンのApify Actorで、SSE経由でMCPサーバーをテストするように設計されており、Authorizationヘッダーをサポートします。
プレーンなJavaScript（旧式スタイル）を使用し、Apifyでホストされているため、セットアップなしで実行できます。

**主な機能:**
- SSE経由で任意のMCPサーバーに接続
- [Apify MCP Server](https://apify.com/apify/actors-mcp-server)と連携して1つ以上のApify [Actors](https://apify.com/store)と相互作用
- コンテキストとユーザークエリに基づいてツールを動的に利用（サーバーがサポートしている場合）

## アプリケーションへのMCPサポート追加

アプリケーションにMCPサポートを追加した場合は、このリストに追加するためにプルリクエストを送信することをお勧めします。MCP統合はユーザーに強力なコンテキストAI機能を提供し、成長するMCPエコシステムの一部となることができます。

MCPサポートを追加する利点:
- ユーザーが独自のコンテキストとツールを持ち込めるようにする
- 相互運用可能なAIアプリケーションの成長するエコシステムに参加
- ユーザーに柔軟な統合オプションを提供
- ローカルファーストAIワークフローをサポート

アプリケーションにMCPを実装するには、[Python](https://github.com/modelcontextprotocol/python-sdk)または[TypeScript SDKドキュメント](https://github.com/modelcontextprotocol/typescript-sdk)を参照してください
